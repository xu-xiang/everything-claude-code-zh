# Bug 修复工作流

## 步骤 1：描述问题
向 Copilot 提供：
- 现象
- 堆栈 / 日志
- 复现步骤
- 预期行为
- 实际行��

## 步骤 2：先分析
要求 Copilot：
1. 总结问题
2. 推测可能根因
3. 标出涉及文件
4. 给出最小修改方案
5. 暂不直接生成代码

## 步骤 3：确认方案
人工确认：
- 修改范围
- 风险点
- 是否影响兼容性 / 生命周期 / 权限 / 性能

## 步骤 4：生成 patch
要求 Copilot：
- 仅修改必要代码
- 保持外部行为不变
- 说明每一处改动的目的

## 步骤 5：验证
要求 Copilot 给出：
- 手动验证步骤
- 测试建议
- 回归点清单
