# Android 系统开发规则

适用于：
- framework
- system service
- Binder / AIDL
- 系统权限与安全相关代码

## 核心规则

1. 优先兼容性和稳定性，避免破坏现有调用方行为。
2. 不轻易修改系统接口语义、返回值、异常行为。
3. 修改 Binder / AIDL 调用链时，必须关注：
   - 权限校验
   - 调用线程
   - 死锁风险
   - 大对象传输风险
   - 跨进程异常传播
4. 修改 system_server 相关代码时，必须关注：
   - 锁顺序
   - 长耗时操作
   - I/O 阻塞
   - 主线程 / handler 线程负载
5. 涉及权限、身份校验、用户隔离、多用户逻辑时，优先保守处理。
6. 不能为了“代码更优雅”而扩大改动面。
7. 不能随意吞异常；需要明确日志、恢复策略和调用方影响。
8. 修改 framework 行为时，应说明兼容性风险和回归点。
9. 涉及系统属性、配置开关、feature flag 时，应保留默认行为。
10. 涉及启动路径、广播、服务注册、ContentProvider 初始化时，要警惕性能回退。

## 输出附加要求

回答时应额外说明：
- 影响的系统组件
- 是否存在权限或稳定性风险
- 如何验证不影响既有流程
