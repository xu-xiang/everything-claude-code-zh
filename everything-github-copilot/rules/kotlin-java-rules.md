# Kotlin / Java 代码规则

## 通用规则

1. 保持与项目现有语言风格一致。
2. Kotlin 优先使用清晰、稳定、可读的写法，而不是炫技写法。
3. Java 代码优先可维护性，不盲目引入复杂函数式风格。
4. 不滥用扩展函数、作用域函数、链式调用。
5. 避免生成过长方法；复杂逻辑拆分时保持命名清晰。
6. 空安全处理要明确，不要用 `!!` 掩盖设计问题。
7. 异常处理应明确，不要静默吞掉错误。
8. 日志要足够定位问题，但不要泄露敏感信息。
9. 若现有模块以 Java 为主，不要为了“现代化”大规模改成 Kotlin。
10. 若现有模块以 Kotlin 为主，新增代码保持 idiomatic，但优先易懂。

## 测试友好性

生成代码时尽量考虑：
- 可注入依赖
- 可 mock
- 低耦合
- 易验证副作用
